-> 置换元素的概念 ( substitution ) 
    - 我们思考 img 标签 是一个 inline 元素 但是为什么 还可以进行设置 width、height 要知道inline元素是无法设置 宽高的
    - 那么 就产生一个 概念 将 img 这类的标签进行单独进行定义啦 那么就是 置换元素
    - 博客定义：一个 内容 不受CSS [视觉格式化] 模型控制，CSS渲染模型并不考虑对此内容的渲染，且元素本身一般拥有固有尺寸（宽度，高度，宽高比）的元素，被称之为置换元素。
    - 自我理解： 我们进行理解 怎么一句话 - 元素拥有固有尺寸 宽高的元素 那么就是置换元素呢 
    - 置换元素的 特点 就是元素是以 inline进行显示的  但是可以进行设置 宽高
        1) 置换元素的 另一个特点
            元素的宽高是 auto 但是元素含有 默认的 固有的宽高
-> 非置换元素的概念
    - 这个就很好理解啦 置换元素 之外的元素都是 非置换元素呢
    - 那么 inline 的非置换元素 是无法进行设置 宽高的 就是 默认的 span 、 em、strong、等等

-> 层叠上下文 ( z-index )
    - 我们都是知道 在CSS中 z-index 设置元素的 层叠次序  但是只能是含有 position 且属性值为 relative 、 absolute、fixed的 如果设置元素 为含有 那么 z-index 将会无效的  
    - 但是我们知道 opacity 会 生成 层叠上下文吗  答案是会进行设置 的 拥有 opacity 的元素 层叠级会重新的进行计算
    - 我们进行 设置 元素的 opacity 样式 那么元素的 层级就会提高 高于 普通的元素
        - 但是需要注意点是 不能设 opacity值为 1  那么就是产生 层叠次序啦
        - 需要设置 opacity 小于 1
    - 我们可以将 z-index 用于图文 替换 
        - 在 背景图片 加载失败时 我们可以将 元素内的文字显示出来
        - 首先设置 显示文字需要使用一个元素 进行包含 
        - 设置 其的 z-index 为 -1 那么在 背景加载失败 就能显示出来啦  背景加载成功 那么就会会覆盖呢
